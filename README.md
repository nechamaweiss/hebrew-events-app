# מערכת תזכורות לאירועים משפחתיים לפי הלוח העברי

מערכת ניהול (למנהל בלבד) לאירועים ושמחות משפחתיים לפי **הלוח העברי**, ששולחת תזכורות בדוא"ל אוטומטית לפני האירוע וביומו.

## טכנולוגיות

- **Next.js 14** (App Router) + **TypeScript** + **Tailwind CSS** — ממשק מלא ב-RTL
- **Prisma ORM** + **SQLite** (ניתן להחלפה ל-PostgreSQL בשינוי שורה אחת)
- **@hebcal/core** — חישובי לוח עברי מדויקים (שנים מעוברות, אדר א'/ב', חודשים חסרים/מלאים)
- **JWT + bcrypt** — אימות מנהל מאובטח (עוגייה httpOnly)
- **Nodemailer** — שליחת מיילים (עם מצב הדמיה כשאין SMTP)
- **node-cron** — מתזמן יומי לתזכורות

## התקנה והרצה

```bash
npm install                # התקנת תלויות
npx prisma migrate dev     # יצירת מסד הנתונים (רץ פעם אחת) + זריעת נתונים
npm run dev                # הרצת השרת → http://localhost:3000
```

### פרטי כניסה ראשוניים (מוגדרים בקובץ `.env`)

- **אימייל:** `admin@example.com`
- **סיסמה:** `Admin1234!`

## מנוע התזכורות (Scheduler)

הרצה כשירות רקע שבודק בכל יום בשעה 08:00 אילו תזכורות לשלוח:

```bash
npm run scheduler          # רץ ברקע, מפעיל את המנוע כל יום ב-08:00
```

הרצה ידנית חד-פעמית לבדיקה (עם תאריך אופציונלי):

```bash
npm run scheduler:once                  # להיום
npm run scheduler:once -- --date=2025-08-31   # לתאריך מסוים
```

## שליחת מיילים אמיתית

כברירת מחדל המערכת עובדת ב**מצב הדמיה** — המיילים מודפסים לקונסול ומתועדים בלוג (ללא שליחה בפועל).
כדי לשלוח מיילים אמיתיים, מלא את פרטי ה-SMTP בקובץ `.env`:

```
SMTP_HOST="smtp.gmail.com"
SMTP_PORT="587"
SMTP_USER="your@email.com"
SMTP_PASS="app-password"
```

## מבנה המערכת

| מסך | נתיב | תיאור |
|-----|------|-------|
| התחברות | `/login` | כניסת מנהל + שחזור סיסמה |
| לוח בקרה | `/` | סטטיסטיקות ואירועים קרובים |
| ניהול אירועים | `/events` | רשימה, חיפוש, סינון, יצירה ועריכה |
| מקבלי התראות | `/recipients` | ניהול נמענים (משויכים למספר אירועים) |
| לוח שנה עברי | `/calendar` | תצוגת חודש עברי עם סימון אירועים |
| לוג מיילים | `/logs` | תיעוד כל המיילים שנשלחו |
| הגדרות העסק | `/settings/business` | שם, לוגו, פרטי שולח, חתימה |
| עיצוב | `/settings/appearance` | צבעי המערכת + מצב בהיר/כהה |

## מעבר ל-PostgreSQL

1. ב-`prisma/schema.prisma` שנה `provider = "sqlite"` ל-`"postgresql"`.
2. ב-`.env` עדכן את `DATABASE_URL` למחרוזת החיבור של PostgreSQL.
3. הרץ `npx prisma migrate dev`.
